# RAG 是什么

## 一句话理解

> **RAG = 让 AI 回答问题之前先"查资料"，而不是只靠训练数据胡编。**

传统 LLM = **闭卷考试**（只靠脑子里记的）
RAG = **开卷考试**（先翻书查资料，再结合资料答题）

## 核心概念

| 概念 | 说明 |
|-------|------|
| **参数化知识** | 训练进模型权重里的"模糊记忆"，更新慢、有幻觉 |
| **非参数化知识** | 存在外部知识库的"精准记忆"，随时可更新 |
| **RAG** | 两者结合：LLM 先从外部知识库检索，再结合检索结果生成答案 |

## RAG 双阶段架构

```
阶段一：检索阶段
  知识向量化（Embedding）→ 存入向量数据库
  用户提问 → 问题向量化 → 相似度搜索 → 召回 Top-K 相关文档片段

阶段二：生成阶段
  相关文档片段 + 用户问题 → 组装 Prompt → LLM 生成 → 带引用的答案
```

## RAG 技术演进三阶段

| | 初级 RAG | 高级 RAG | 模块化 RAG |
|---|---|---|---|
| **流程** | 离线索引 → 在线检索 → 生成 | 离线索引 → *检索前优化* → 检索 → *检索后优化* → 生成 | 积木式可编排 |
| **关键技术** | 基础向量检索 | 查询重写(Query Rewrite) / 结果重排(Rerank) | 动态路由 / 查询转换 / 多路融合 |
| **局限性** | 效果不稳定 | 流程相对固定 | 系统复杂性高 |

## 技术选型路径

```
提示词工程（不改模型、不改上下文）
    ↓  模型缺特定知识
RAG（不改模型、引入外部知识库）
    ↓  目标是改变行为/风格
微调 SFT（改模型权重）
```

选型原则：**优先选对模型改动最小，成本最低的方案。**
